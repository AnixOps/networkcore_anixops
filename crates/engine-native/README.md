# engine-native

`engine-native` contains the first native proxy execution engine adapter contracts.

The crate implements `ProxyEngineService` with stable descriptor, structured listener/node/route graph validation, lifecycle, status, and event diagnostics for the future in-process native engine. It also exposes the first runtime handle contract for config-driven runtime assembly planning, real loopback TCP listener binding and release, a controlled loopback TCP accept loop, accepted connection counting, pre-protocol connection close diagnostics, SOCKS5 greeting version/auth-method read diagnostics, SOCKS5 no-auth method selection and unsupported-auth rejection diagnostics, SOCKS5 auth method response write diagnostics, SOCKS5 command header read and unsupported-command rejection diagnostics, SOCKS5 CONNECT target address read diagnostics, SOCKS5 route/outbound behavior selection diagnostics, SOCKS5 outbound CONNECT request frame generation diagnostics, SOCKS5 outbound TCP connection plan diagnostics, SOCKS5 outbound TCP connection attempt diagnostics, SOCKS5 route/outbound unwired rejection diagnostics, SOCKS5 CONNECT failure response write diagnostics, SOCKS outbound handler handoff, startup failure release reports, native runtime events, and foreground lifecycle handoff status. `NativeProxyEngineService::start` still rejects with a stable unavailable diagnostic until those resources are connected to SOCKS5 outbound connection handling and data relay.

The accept loop only accepts loopback TCP connections, reads the first SOCKS5 greeting version/auth-method bytes when present, selects no-auth or rejects unsupported auth methods for diagnostics, writes the SOCKS5 method response, reads the SOCKS5 command header after no-auth selection, reports unsupported commands, reads CONNECT IPv4/domain/IPv6 target addresses, selects the configured SOCKS outbound handler for the CONNECT target, generates the upstream SOCKS5 CONNECT request frame diagnostics in memory, creates an in-memory outbound TCP connection plan, attempts a bounded TCP connection to IP-literal SOCKS outbound endpoints, reports route/outbound as unwired, writes a SOCKS5 CONNECT general failure response, and then closes the accepted connection before outbound request writing or data relay. It does not implement BIND/UDP ASSOCIATE execution, outbound CONNECT request writing, outbound data plane, UDP, TUN, DNS, MITM, daemon control, platform mutation, SOCKS5 success response writing, or `networkcore-linux start` wiring.

Verification for this crate is performed only by GitHub Actions, following the repository CI/CD policy.
